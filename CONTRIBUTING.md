# Contributing to Rancher DevOps Operator

Thank you for your interest in contributing! Please follow these guidelines to help us review your changes quickly and smoothly.

## How to Contribute
- Fork the repository and create your branch from `main`.
- If you've fixed a bug or added code, add tests if possible.
- Ensure your code passes CI/CD and linting.
- Open a Pull Request with a clear description of your changes.

## Code Style
- Use clear, descriptive commit messages.
- Follow .NET and Kubernetes best practices.
- Keep PRs focused and minimal.

## Reporting Issues
- Use the issue template for bug reports and feature requests.
- Provide as much detail as possible (logs, steps to reproduce, etc).

## Community
- Be respectful and inclusive.
- See our [Code of Conduct](CODE_OF_CONDUCT.md).
